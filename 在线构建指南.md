## 方案1：GitHub Actions（推荐，完全免费）

### ✨ 优点
- ✅ 完全免费
- ✅ 自动构建
- ✅ 每次提交代码自动生成 APK
- ✅ 可以下载历史版本

### 📋 使用步骤

#### 第一步：注册 GitHub 账号
1. 访问：https://github.com/
2. 点击 "Sign up" 注册账号（免费）
3. 验证邮箱

#### 第二步：创建仓库
1. 登录 GitHub
2. 点击右上角 "+" → "New repository"
3. 填写信息：
   - Repository name: `daily-plan-app`
   - Description: `日常计划应用`
   - 选择 "Public"（公开）或 "Private"（私有）
4. 点击 "Create repository"

#### 第三步：上传代码

**方式A：使用 GitHub Desktop（推荐）**
1. 下载 GitHub Desktop：https://desktop.github.com/
2. 安装并登录
3. 点击 "Add" → "Add existing repository"
4. 选择文件夹：`D:\我的文档\日常计划Flutter应用`
5. 点击 "Publish repository"

**方式B：使用命令行**
```bash
cd "D:\我的文档\日常计划Flutter应用"
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/daily-plan-app.git
git push -u origin main
```

**方式C：网页上传（最简单）**
1. 在 GitHub 仓库页面，点击 "uploading an existing file"
2. 将整个项目文件夹拖拽到页面
3. 点击 "Commit changes"

#### 第四步：自动构建
1. 代码上传后，GitHub Actions 会自动开始构建
2. 点击仓库的 "Actions" 标签查看构建进度
3. 等待约 5-10 分钟，构建完成

#### 第五步：下载 APK
1. 构建完成后，点击 "Actions" → 选择最新的构建
2. 在 "Artifacts" 部分，点击 "app-release" 下载
3. 解压 ZIP 文件，得到 `app-release.apk`
4. 传输到手机安装

### 🔄 后续使用
每次修改代码并提交，GitHub Actions 会自动构建新的 APK！

## 📝 GitHub Actions 详细教程

### 准备工作 ✅
我已经为您创建了 GitHub Actions 配置文件：
```
.github/workflows/build.yml
```

### 第一步：注册 GitHub

1. 打开浏览器，访问：**https://github.com/**
2. 点击右上角 **"Sign up"**
3. 填写信息：
   - Email: 您的邮箱
   - Password: 设置密码
   - Username: 用户名（如：dailyplan2026）
4. 验证邮箱
5. 完成注册

### 第二步：创建仓库

1. 登录 GitHub
2. 点击右上角 **"+"** → **"New repository"**
3. 填写：
   - Repository name: `daily-plan-app`
   - Description: `日常计划Flutter应用`
   - 选择 **"Public"**（推荐）
4. **不要**勾选 "Add a README file"
5. 点击 **"Create repository"**

### 第三步：上传代码（网页上传最简单）

1. 在新创建的仓库页面，点击 **"uploading an existing file"**
2. 打开文件夹：`D:\我的文档\日常计划Flutter应用`
3. 选择所有文件和文件夹（**Ctrl+A**）
4. 拖拽到 GitHub 网页
5. 等待上传完成
6. 在底部输入提交信息：`Initial commit`
7. 点击 **"Commit changes"**

### 第四步：查看构建进度

1. 上传完成后，点击仓库顶部的 **"Actions"** 标签
2. 会看到一个正在运行的工作流（🟠 橙色圆圈）
3. 点击进入查看详细进度
4. 等待约 **5-10 分钟**，直到显示 ✅ 绿色勾号

### 第五步：下载 APK

#### 方式A：从 Artifacts 下载
1. 在 Actions 页面，点击完成的构建
2. 滚动到底部 **"Artifacts"** 部分
3. 点击 **"app-release"** 下载（ZIP 文件）
4. 解压 ZIP，得到 `app-release.apk`

#### 方式B：从 Releases 下载（推荐）
1. 点击仓库右侧的 **"Releases"**
2. 会看到自动创建的 Release
3. 在 **"Assets"** 下直接下载 `app-release.apk`

### 第六步：安装到手机

1. 将 `app-release.apk` 发送到手机
   - 微信发给自己
   - QQ 文件助手
   - USB 数据线传输
2. 在手机上点击 APK 文件
3. 允许"安装未知应用"
4. 点击"安装"
5. 完成！🎉

## ❓ 常见问题

### Q1: GitHub Actions 构建失败？
**A:** 检查：
1. 代码是否完整上传
2. `.github/workflows/build.yml` 文件是否存在
3. 查看错误日志

### Q2: 无法下载 Artifacts？
**A:** 
- 需要登录 GitHub 账号
- 或者从 Releases 页面下载

### Q3: 构建时间太长？
**A:** 
- 首次构建需要 10-15 分钟
- 后续构建会更快（5-8 分钟）
- 这是正常的

### Q4: 想要修改代码后重新构建？
**A:** 
1. 修改本地代码
2. 重新上传到 GitHub
3. 自动触发新的构建

### Q5: 可以私有仓库吗？
**A:** 可以！免费账号支持私有仓库
